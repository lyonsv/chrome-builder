## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, status indicator margin |
| md | 16px | Default container padding, section margins |
| lg | 24px | Section breaks |
| xl | 32px | Major layout gaps |
| 2xl | 48px | Not used in popup (400px width constraint) |
| 3xl | 64px | Not used in popup (400px width constraint) |

Exceptions:
- Picker highlight border: 2px outline (not a spacing value — border width only)
- Picker label tag: 4px 8px padding (xs/sm combination — existing badge pattern from `.service-category`)
- Touch targets (picker cursor mode): minimum 44px interactive area on the inspected page — enforced via the overlay, not the popup itself

Source: existing `css/popup.css` spacing values extracted and normalized to the 8-point scale.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.4 |
| Label | 12px | 400 (regular) | 1.3 |
| Heading (h3) | 16px | 600 (semibold) | 1.3 |
| Display (h1) | 18px | 600 (semibold) | 1.2 |

Two weights only: `400 (regular)` for body and label text; `600 (semibold)` for headings and display. Body vs label distinction is made by size (14px vs 12px), not weight.

Source: `css/popup.css` — `body` (14px/1.4), `.current-url` (12px), `.section h3` (16px/600), `.header h1` (18px/600). No new type sizes introduced in Phase 3.

## Color

Source: `css/popup.css` — full color inventory extracted. No new colors introduced in Phase 3; picker accent reuses existing `#2196f3`.

## Component Inventory

New UI elements introduced in Phase 3 (extends existing popup structure):

### 1. "Pick Element" Button
- **Class**: `.btn .btn-pick-element`
- **Idle state**: Secondary style — background `#6c757d`, white text, label "Pick Element"
- **Active/picker-mode state**: Accent style — background `#2196f3`, white text, label "Picking… (Esc to cancel)"
- **Placement**: Inside the Actions section, above the existing "Start Analysis" button
- **Width**: Full-width (`width: 100%`) — same as `.btn`
- **Disabled**: When analysis is running (same `.btn:disabled` rule, opacity 0.5)

### 2. Selection Summary Panel
- **Appears**: After element is locked in via picker; replaces/augments the current-url area
- **Class**: `.selection-summary`
- **Surface**: White card — same as `.options` (background `#fff`, border `1px solid #ddd`, border-radius 8px, padding 12px 16px)
- **Contents** (three rows):
  - Row 1: Scope badge (`div.header-nav`) — monospace label, `#2c3e50` text
  - Row 2: Child count — muted label (`#666`) + value (`#2c3e50`), semibold
  - Row 3: Clear button — right-aligned, text style, color `#6c757d`, label "× Clear"
- **Indicator dot**: 8px filled circle, color `#2196f3`, left of the scope badge (reuses `.status-indicator` shape without animation)
- **No animation on indicator dot** — static, not pulsing (selection is locked, not in-progress)

### 3. Picker Overlay (injected into inspected page — NOT in popup)
- **z-index**: 2147483647 (max safe integer — ensures overlay is always on top)
- **Highlight box**: `outline: 2px solid #2196f3` following cursor target element (outline not border — avoids layout shift)
- **Label chip**: Positioned top-left of highlight box, content = `tag.primaryClass` (e.g. `div.header-nav`)
  - Background: `#2196f3`, white text, padding 2px 6px, border-radius 4px, font-size 11px
  - Same visual as `.service-category` chip in the popup
- **Overlay background**: Transparent — overlay only draws the highlight box, no page dimming
- **Cleanup**: Overlay element removed from DOM on: (a) click to select, (b) Escape key, (c) popup close/unload
- **Cursor**: `crosshair` on the overlay layer
- **Escape key**: Cancels picker, removes overlay, popup state returns to previous (no selection change)

### 4. Scope Indicator in Status Bar
- **When element selected**: Status bar text changes from "Ready to analyze" to "Scoped: div.header-nav"
- **Status bar background/border**: No change — retains existing `#e3f2fd` / `#2196f3` left-border style
- **When cleared or no selection**: Status bar reverts to "Ready to analyze" idle text

## Interaction States

### "Pick Element" Button — State Machine

| State | Visual | Label |
|-------|--------|-------|
| Idle (no selection) | `.btn-secondary` — `#6c757d` bg | "Pick Element" |
| Picker active | `.btn-primary` — `#2196f3` bg | "Picking… (Esc to cancel)" |
| Element locked | `.btn-secondary` — `#6c757d` bg | "Re-pick Element" |
| Disabled (analysis running) | opacity 0.5, cursor not-allowed | "Pick Element" (grayed) |

### Selection Summary Panel — Visibility

| Condition | Panel Visibility |
|-----------|-----------------|
| No element selected | Hidden (`display: none`) |
| Picker active (mid-selection) | Hidden — popup may be minimized during picker |
| Element locked in | Visible, shows tag + class + child count + clear button |
| "× Clear" clicked | Hidden, scope returns to full-page |

### Analysis Button — Scope Awareness

| Scope State | "Start Analysis" Label |
|-------------|----------------------|
| Full-page (no selection) | "Start Analysis" (existing) |
| Element scoped | "Analyze Selected Element" |

No other button labels change.

## Implementation Notes for Executor

1. **All new CSS goes in `css/popup.css`** — append new rules after existing styles. No new CSS files.
2. **`.btn-pick-element` state toggling** — add/remove `.active` class via `popup.js`. Do not create a separate button element for each state; toggle class and update `textContent`.
3. **Selection summary** uses existing card patterns — copy the structure of `.details-section` (white bg, 1px ddd border, 8px radius, 12px 16px padding).
4. **Picker overlay is injected via `chrome.scripting.executeScript`** — it is NOT part of `popup.html`. The overlay lives entirely in the inspected page's DOM and must be fully removed on exit (no leaked styles or DOM nodes).
5. **`#statusText` content** — Phase 3 adds scope-aware status messages. All existing idle/analyzing/error messages remain unchanged; scope messages are additive.
6. **Monospace for selector display** — `tag.class` strings in the selection summary and picker label chip should use `font-family: monospace` to differentiate code from prose.
