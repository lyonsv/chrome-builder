  <files>content.js</files>
  <behavior>
    - buildSignature(el) returns "tagname.class-a.class-b" (classes sorted alphabetically) for classed elements
    - buildSignature(el) returns "tagname" for elements with no classes
    - Identical signatures (same tag + same sorted classes) appear only once in the elements map
    - occurrences is incremented for each duplicate signature instead of creating a new entry
    - Each element entry has: styles (object of ~60 CSS properties), occurrences (integer), exampleHtml (outerHTML truncated to 500 chars)
    - Elements with active ::before or ::after (content !== 'none' && content !== 'normal') have a pseudos key
    - globals contains: tokens (empty object for now â€” Plan 02 fills this), body (DESIGN_SYSTEM_PROPERTIES from document.body), html (font-size and font-family from documentElement)
    - computedStyles is returned as a top-level key from analyzeWebsite()
    - states key exists on each element entry (empty object for now â€” Plan 02 fills this)
  </behavior>
  <action>
    Add the following to content.js inside the WebsiteAnalyzer class, before the closing `}` of the class:

    1. **Constants** (at top of the file, outside the class, near the existing CHUNK_ constants):
       Add `DESIGN_SYSTEM_PROPERTIES` array (62 properties from RESEARCH.md canonical list) and `PSEUDO_ELEMENT_PROPERTIES` array (20 properties).

    2. **`buildSignature(el)` method** on the class:
       - `const tag = el.tagName.toLowerCase()`
       - `const classes = [...el.classList].sort().join('.')`
       - Return `classes ? \`${tag}.${classes}\` : tag`

    3. **`_extractPseudoStyles(el)` method** on the class:
       - For each of `['::before', '::after']`, call `window.getComputedStyle(el, pseudo)`
       - Read `content` property; skip if `content === 'none'` or `content === 'normal'` or falsy
       - If active, capture all PSEUDO_ELEMENT_PROPERTIES values using `.getPropertyValue(prop).trim()`
       - Return the pseudos object, or `undefined` if no active pseudo-elements

    4. **`_buildGlobalSection()` method** on the class:
       - Read `document.body` computed style for all DESIGN_SYSTEM_PROPERTIES
       - Read `document.documentElement` computed style for `font-size` and `font-family`
       - Return `{ tokens: {}, body: bodyStyles, html: { fontSize, fontFamily } }`
       - Note: `tokens` is populated by Plan 02's `_buildTokenVocabulary()` â€” leave as empty object here

    5. **`extractComputedStyles()` method** on the class:
       - Call `this._buildGlobalSection()` for globals
       - Walk `document.querySelectorAll('*')` with a `Map` keyed by signature
       - For each element: compute signature with `buildSignature(el)`. If seen: increment `occurrences`, continue.
       - For first occurrence: call `window.getComputedStyle(el)`, read all DESIGN_SYSTEM_PROPERTIES, call `_extractPseudoStyles(el)`
       - Build entry: `{ styles, pseudos (if present), states: {}, occurrences: 1, exampleHtml: el.outerHTML.slice(0, 500) }`
       - Return `{ globals, elements: Object.fromEntries(seen), crossOriginStylesheets: [] }`

    6. **Wire into `analyzeWebsite()`**:
       - Add `this.extractComputedStyles()` to the `Promise.all([...])` array â€” but since it's synchronous internally, wrap: `Promise.resolve(this.extractComputedStyles())`
       - Capture result: `const computedStylesResult = await Promise.resolve(this.extractComputedStyles())`
       - Actually: call it after the Promise.all (it's CPU-bound synchronous, not I/O â€” call sequentially after the parallel block to avoid blocking other tasks)
       - Add `computedStyles: this.extractComputedStyles()` call inline: after the `await Promise.all([...])` block, call `const computedStyles = this.extractComputedStyles()` then include `computedStyles` in the return object

    **Anti-patterns to avoid (per RESEARCH.md):**
    - Do NOT call `getComputedStyle` before checking the signature map â€” signature check must come first
    - Do NOT use `rule.type === CSSRule.STYLE_RULE` â€” use `instanceof CSSStyleRule`
    - Do NOT spread large Uint8Arrays â€” not relevant here, but note for reference
    - For elements with no classes, the bare `tagname` signature means many structurally different elements share it. Accept this for now â€” the deduplication is correct per the spec (tag-only is a valid signature per CONTEXT.md decisions)
