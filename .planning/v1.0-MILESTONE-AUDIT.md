## Integration Findings

### Issue A — High: Popup result counts always display 0

**Location:** `popup.js:286`
**Affected requirements:** TRACK-01 (display), SCOPE-01, SCOPE-03 (feedback)

The content script's `ANALYZE_WEBSITE` handler sends `{ success: true }` after routing analysis data via `sendChunked` to the background. Popup's `analyzeWebsiteContent()` resolves `response.data` — always `undefined`. So `this.analysisData` in the popup contains only `{ networkRequests, screenshot, options, timestamp }`.

**Consequence matrix:**

| Display element | Expected | Actual |
|----------------|----------|--------|
| Tracking Events count | dataLayer.length | 0 |
| Assets count | assetUrls.length | 0 |
| Services count | thirdPartyServices.length | 0 |
| FETCH_ASSETS call | explicit with progress | skipped (background fallback fires) |

**Impact on ZIP:** None — background assembles ZIP from its own stored `analysisData`. ZIP content is correct.
**Impact on UX:** All popup result counts are incorrect.

**Fix direction:** After background stores the complete analysis (after chunked transfer completes), push a `ANALYSIS_STORED` message to the popup with the data, or have popup request data from background via `GET_ANALYSIS`.

---

### Issue B — Medium: Hardcoded site names in popup.js fallback

**Location:** `popup.js:751-786` (`detectServicesForKnownSites`)
**Affected requirements:** TRACK-03

Fallback analysis path contains explicit hostname matches (`github.com`, `amazon.com`, `netflix.com`, `example-ecommerce.com`). Fires when content script injection fails. TRACK-03 requires all detection to use generic observable patterns.

**Fix direction:** Replace hostname lookup table with generic signal-based detection (same pattern as `content.js:identifyThirdPartyServices()`), or remove the fallback entirely and show "detection unavailable" when content script fails.

---

### Issue C — Low: css/ directory empty in ZIP

**Location:** `background.js:688`
**Affected requirements:** SCOPE-02

`fileTree['css/'] = {}` — scaffolded but no stylesheet content written. Raw CSS text from fetched stylesheets (if captured) is not placed here.

**Fix direction:** Write fetched CSS file text into `css/` using filenames derived from the stylesheet URL, or document that `css/` is intentionally reserved for a future phase.
