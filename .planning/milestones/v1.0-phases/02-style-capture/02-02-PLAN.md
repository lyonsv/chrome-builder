  <behavior>
    - _extractPseudoClassRules() returns { pseudoMap: { signature: { ':hover': { prop: value }, ... } }, crossOriginUrls: [] }
    - Only processes rules whose selectorText ends with a known pseudo-class (':hover', ':focus', ':focus-visible', ':active', ':disabled') — uses endsWith(pseudo) check to avoid fragile complex-selector stripping
    - For matched rules, strips the pseudo suffix to get base selector, wraps querySelectorAll(baseSelector) in try/catch for invalid selectors
    - Only attributes rules to signatures whose base selector matches at least one live element
    - Recurses into @media, @supports, @layer (rule.cssRules check for nesting)
    - Cross-origin sheets (SecurityError on cssRules access) are caught and their sheet.href pushed to crossOriginUrls
    - Uses instanceof CSSStyleRule (NOT rule.type === CSSRule.STYLE_RULE which is deprecated)
    - _buildTokenVocabulary() returns { color: {}, spacing: {}, font: {}, other: {} } grouped by --color-*, --spacing-*, --font-* prefixes

    - usedBy is populated by scanning all CSSStyleRule declarations for val.match(/var\((--[^,)]+)/) patterns — when found, the rule's matching live-element signatures are added to the token's usedBy array
    - Token values always have .trim() applied (per Pitfall 5 in RESEARCH.md — leading whitespace is spec-compliant)
    - extractComputedStyles() is updated to call both methods and merge their results:
      - globals.tokens = this._buildTokenVocabulary()
      - crossOriginStylesheets populated from pseudoClassResult.crossOriginUrls
      - Each element entry's states merged from pseudoClassResult.pseudoMap[signature] (or {} if not found)
  </behavior>
  <action>
    Add two methods to the WebsiteAnalyzer class in content.js, then update extractComputedStyles() to call them.

    **Method 1: `_extractPseudoClassRules()`**

    Implementation:
    ```
    const PSEUDO_CLASSES = [':hover', ':focus', ':focus-visible', ':active', ':disabled'];
    const pseudoMap = {};
    const crossOriginUrls = [];

    function walkRules(rules) {
      for (const rule of rules) {
        if (rule instanceof CSSStyleRule) {
          for (const pseudo of PSEUDO_CLASSES) {
            if (rule.selectorText.endsWith(pseudo)) {
              // Strip the pseudo suffix to get base selector
              const baseSelector = rule.selectorText.slice(0, -pseudo.length).trim();
              if (!baseSelector) continue;
              try {
                const matches = document.querySelectorAll(baseSelector);
                for (const el of matches) {
                  const sig = this.buildSignature(el);
                  if (!pseudoMap[sig]) pseudoMap[sig] = {};
                  if (!pseudoMap[sig][pseudo]) pseudoMap[sig][pseudo] = {};
                  for (let i = 0; i < rule.style.length; i++) {
                    const prop = rule.style[i];
                    pseudoMap[sig][pseudo][prop] = rule.style.getPropertyValue(prop).trim();
                  }
                }
              } catch (_) { /* invalid selector after strip — skip */ }
            }
          }
        }
        if (rule.cssRules) walkRules(rule.cssRules); // recurse @media/@supports/@layer
      }
    }

    for (const sheet of document.styleSheets) {
      try { walkRules(sheet.cssRules || []); }
      catch (_) { if (sheet.href) crossOriginUrls.push(sheet.href); }
    }
    return { pseudoMap, crossOriginUrls };
    ```

    Note: `walkRules` references `this.buildSignature` — use an arrow function or bind correctly so `this` is the WebsiteAnalyzer instance. Best approach: make `walkRules` an inner arrow function that captures `this` from the method scope.

    **Method 2: `_buildTokenVocabulary()`**

    Steps:
    1. Read all `:root` custom properties from `window.getComputedStyle(document.documentElement)` — iterate `rootStyle.length`, check `rootStyle[i].startsWith('--')`, store `{ value: rootStyle.getPropertyValue(name).trim(), definedAt: ':root', usedBy: [] }`
    2. Scan all accessible CSSStyleRule declarations for scoped custom property definitions (prop.startsWith('--') and selectorText !== ':root') — update tokens map with `scopedAt` and `value`
    3. Scan all accessible CSSStyleRule declarations for `var(--` usage in property values — use `val.match(/var\((--[^,)]+)/)` to extract token name; find live elements matching rule's base selector (same querySelectorAll + try/catch pattern), get their signatures, push to token's `usedBy` array (deduplicate with Set or includes check)
    4. Group into `{ color: {}, spacing: {}, font: {}, other: {} }` by prefix: `--color-*` → color, `--spacing-*` → spacing, `--font-*` → font, everything else → other
    5. Return grouped object

    **Update `extractComputedStyles()`**:
    - After computing the elements map, call `const pseudoClassResult = this._extractPseudoClassRules()`
    - Merge states: for each signature in the elements map, set `entry.states = pseudoClassResult.pseudoMap[sig] || {}`
    - Set `globals.tokens = this._buildTokenVocabulary()`
    - Set `crossOriginStylesheets = pseudoClassResult.crossOriginUrls`
    - Return `{ globals, elements: Object.fromEntries(seen), crossOriginStylesheets }`

    **Anti-patterns to avoid:**
    - Do NOT use `.replace(pseudo, '')` for base selector extraction — use `.slice(0, -pseudo.length)` which is safe for suffix-only matching (per Pitfall 3 in RESEARCH.md)
    - Do NOT assume getComputedStyle returns `var(--token)` for standard properties — it does NOT; token names come from stylesheet rule text only
    - Do NOT forget try/catch on every `sheet.cssRules` access — SecurityError on cross-origin sheets
    - Always `.trim()` custom property values (Pitfall 5)
