## Scope of UI Changes in Phase 4

Phase 4 is primarily a data pipeline phase. The only popup UI surface change is:

- **New result row** in the existing `#resultsSection` `.results` list: "Tracking Events:" with a count value (consistent with Assets Found, Network Requests, Frameworks Detected, Third-Party Services rows already present)
- **No new sections, buttons, panels, or overlays** — the tracking capture is automatic and runs as part of every analysis

All other phase deliverables (`tracking/events.json`, `tracking/schema.json`, index.json summary) are ZIP output artifacts — not UI surfaces.

|-------|-------|-------|

## Interaction Contract

### Existing interactions unchanged

All existing popup interactions (Pick Element, Start Analysis, Download Package, checkboxes) are unaffected by Phase 4.

### New interaction: Tracking result row

- Renders in `#resultsSection` `.results` list, after "Third-Party Services:" row
- Element: `<div class="result-item">` containing `<label>Tracking Events:</label>` and `<span id="trackingCount">-</span>`
- Initial state: `-` (dash, same as all other result items before analysis runs)
- Post-analysis state: numeric string (e.g. `"42"` or `"0"`)
- No click behavior — display only, consistent with all other result rows
- `popup.js` sets `document.getElementById('trackingCount').textContent` from `analysisResult.trackingData.dataLayer.length` (or `0` if absent)

### State machine for tracking count display

| Condition | Display Value |
|-----------|---------------|
| Before analysis | `-` |
| Analysis running (spinner visible) | `-` (unchanged during analysis) |
| Analysis complete, dataLayer found | `{N}` (integer string) |
| Analysis complete, no dataLayer | `0` |
| Analysis complete, trackingData absent | `0` (defensive fallback) |

## Implementation Notes for Executor

1. **Only one HTML change**: Add one `<div class="result-item">` inside the `#resultsSection .results` div in `popup.html`. No new sections, no new CSS classes needed — `.result-item`, `label`, and `span` styles are already defined.

2. **Only one JS change in popup.js**: After the existing result count updates (assetsCount, networkCount, frameworksCount, servicesCount), add: `document.getElementById('trackingCount').textContent = result.trackingData ? String(result.trackingData.dataLayer.length) : '0';`

3. **No CSS changes**: All required styles already exist in `css/popup.css`.

4. **All other Phase 4 work is data pipeline**: `content.js` (captureTrackingData method) and `background.js` (deriveEventSchema helper, downloadAsZip modifications) — no visual surfaces.
